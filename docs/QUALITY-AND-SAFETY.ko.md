# 품질과 안전

[English](QUALITY-AND-SAFETY.md)

이 문서는 PCssak Biuja의 안전 모델과 공개 배포 전에 필요한 검증을 설명합니다. 개발 의도와
과정의 근거이지, 오류가 전혀 없다거나 되돌리기로 모든 외부 파일시스템 변경을 복구할 수 있다는
약속이 아닙니다.

## 파일 작업 안전 모델

PCssak Biuja는 사용자가 확인할 수 있는 다음 순서를 중심으로 설계됐습니다.

1. 사용자가 작업할 범위를 선택합니다.
2. 앱이 지원되는 항목을 확인하고 정리 계획을 준비합니다.
3. 사용자가 변경 전에 미리보기를 검토합니다.
4. 네이티브 파일 작업 경계에서 각 지원 작업에 필요한 조건을 다시 확인합니다.
5. 기존 대상 파일을 의도적으로 덮어쓰지 않고 사용자 파일을 영구 삭제하지 않습니다.
6. 완료된 지원 작업을 로컬 작업저널에 기록합니다.
7. 현재 파일시스템 상태가 다른 파일을 덮어쓰지 않는 안전한 역작업을 허용할 때만 저널을
   사용해 되돌립니다.

경로·식별 정보·권한·대상·현재 상태가 더 이상 안전하지 않으면 추측하기보다 거부하는 것이
원칙입니다. 이 모델은 위험을 줄이지만 여러 파일 작업을 원자적 트랜잭션이나 백업으로 만들지는
않습니다.

## 미리보기는 판단 단계입니다

미리보기는 PCssak Biuja가 변경 전에 어디로 무엇을 이동하려는지 보여주기 위한 장치입니다.
사용자는 분류·원본·대상 정보를 확인하고 중요하지 않은 작은 폴더에서 시작해야 합니다. 다른
앱, 동기화 서비스나 사용자가 미리보기 뒤 파일시스템을 바꾸면 계획이 오래된 상태가 될 수
있습니다.

따라서 실행 단계는 미리보기를 무조건적인 권한이 아니라 제안으로 다뤄야 합니다. 변경된 원본,
새로 사용 중인 대상, 사라진 드라이브, 부족한 권한이나 잠긴 파일은 덮어쓰기 대신 안전한 실패나
거부로 처리해야 합니다.

## 덮어쓰기 금지와 영구 삭제 금지

- PCssak Biuja는 기존 대상 파일을 의도적으로 대체하지 않습니다.
- 정리 작업에서 영구 삭제를 수행하지 않으며 지원되는 정리 작업은 Windows 휴지통을 사용합니다.
- 이름·대상 충돌은 확인할 수 있게 표시하거나 안전하게 거부합니다.
- 이 보호는 PCssak Biuja가 수행한 작업에 적용되며 클라우드 공급자, 다른 앱, 악성코드,
  하드웨어 장애나 사용자의 직접 작업을 통제할 수는 없습니다.

보안 삭제 제품이 아니며, “영구 삭제 금지”를 앱 밖의 원인으로 데이터가 손실될 수 없다는
보장으로 해석하면 안 됩니다.

## 작업저널과 되돌리기

로컬 작업저널은 완료된 지원 작업을 기록합니다. 되돌리기는 이 기록을 사용하며 이전 배치를
복원한다는 이유로 현재 존재하는 경로를 덮어쓰면 안 됩니다.

되돌리기는 현재 조건이 허용하는 범위에서만 가능합니다. 기록된 항목이 변경·교체·삭제·이름
변경·잠김·외부 동기화됐거나 저장장치를 사용할 수 없으면 거부될 수 있습니다. 작업저널은
작업에 대한 메타데이터이며 파일의 두 번째 사본이 아닙니다. 휴지통 작업은 해당 항목이 Windows
휴지통에 남아 있을 때만 복원할 수 있습니다. 대체할 수 없는 데이터에는 별도의 검증된 백업이
필요합니다.

## 로컬 우선 개인정보 경계

- 파일 이름·경로·정리 판단·미리보기 생성·작업저널·되돌리기는 기기에서 처리됩니다.
- PCSSAK은 정리 서비스를 제공하기 위해 사용자 파일이나 파일 내용을 받지 않습니다.
- 공개 무료 Early Access에는 PCSSAK 계정, 광고, 사용 분석, 추적, 클라우드 파일 색인,
  자동 오류 업로드가 없습니다.
- 공개 Issue는 이 로컬 경계 밖입니다. 게시 전에 개인 경로·파일 이름·폴더 목록·고객 자료·
  비밀정보·문서 내용을 제거해야 합니다.

## 제한된 네트워크 예외

로컬 처리 안내에는 다음 두 가지 전달 예외가 있습니다.

- 독립형 빌드는 PCSSAK Tauri 업데이트 키로 서명된 업데이트를 확인하고 내려받기 위해 공식
  `pcssakinc` GitHub 릴리스에 연결할 수 있습니다.
- 필수 Microsoft Edge WebView2 Runtime이 없거나 복구가 필요하면 Windows 또는 설치
  프로그램이 Microsoft의 WebView2 배포 서비스에 연결할 수 있습니다.

어느 예외도 파일 내용을 올리거나 분석하는 PCSSAK 서비스가 아닙니다. GitHub 업데이트 확인을
사용할 수 없어도 핵심 파일 정리는 사용할 수 있어야 합니다.

## 업데이트와 배포 무결성

배포 과정은 다음 항목을 요구하도록 설계합니다.

- JavaScript·Tauri·Rust·업데이트 메타데이터의 앱 버전 일치
- 의존성 잠금 파일과 포함된 제3자 라이선스·고지문 검토
- 해당 릴리스가 지원한다고 표시한 경우 아키텍처별 x64·x86 빌드 파일
- 업데이트 파일의 PCSSAK Tauri 서명
- 최종 파일에서 생성한 `latest.json`, 분리 서명 파일과 `SHA256SUMS.txt`
- 공개 파일 크기·해시와 로컬에서 승인한 파일 비교
- 개발 빌드를 실수로 올리지 않도록 하는 명시적 배포 승인 단계
- 개인키·서명 비밀번호·토큰·관계없는 내부 파일의 추적 거부

업데이트 서명은 앱 내부 업데이트 경로를 보호합니다. 설치 파일을 Authenticode 서명 상태로
만드는 것은 아닙니다. 최초 설치 파일은 Windows 게시자 수준에서 미서명 상태이므로 알 수 없는
게시자나 SmartScreen이 나타날 수 있습니다. 사용자는 SHA-256을 확인하고 Windows 보안 장치를
끄지 않아야 합니다.

## 검증 계층

공개 빌드는 해당 버전의 결과를 기록하고 검토한 뒤에만 배포해야 합니다. 관련 검증 계층은 다음과
같습니다.

- 정리 계획·충돌·작업저널·되돌리기·오래된 상태 회귀 테스트
- TypeScript 형식 검사와 프런트엔드 프로덕션 빌드
- Rust 테스트·포맷·린트와 네이티브 파일 경계 검사
- 지원 언어 전체의 번역 완전성과 자리표시자 일치 검사
- 문서화한 Windows 11 Home/Pro x64 대상의 깨끗한 x64 빌드와 설치 앱 실행 점검
- 릴리스가 Windows 10 호환성을 표시한 경우 별도의 최선 노력 베타 점검
- x86 파일을 제공하기 전 32비트 Windows 10과 Windows x64 호환 모드에서 x86 빌드·베타
  실행 점검
- 설치 파일·업데이트 명세·서명·체크섬·업그레이드·되돌리기 경로 점검
- 지원 범위 안에서 잠긴 파일, 권한, 이미 사용 중인 대상, 외부 변경, 클라우드·이동식 저장장치
  경고와 중단된 다중 파일 작업 시험

정확한 테스트 개수와 성능 결과는 날짜가 있는 배포 기록에 남깁니다. 영구적인 사실처럼 이
문서에서 광고하지 않으며 자동 테스트 통과가 실제 설치 앱 점검을 대신하지 않습니다.

## 호환성 주장

현재 지원 중인 Windows 11 Home/Pro x64가 권장 환경이자 공식 검증 예정 대상입니다. Windows
11은 64비트 전용이며 32비트 에디션이 없습니다. x86 설치기는 32비트 Windows 10 실행과
Windows x64의 32비트 앱 호환 시험을 위한 범위가 좁은 베타 파일이며 Windows 11 32비트
빌드가 아닙니다.

Windows 10 22H2 x64·x86은 제한적·최선 노력 베타 호환성만 제공합니다. Microsoft는
[Windows 10 일반 지원을 2025년 10월 14일 종료했고](https://support.microsoft.com/en-us/windows/deployment/updates-lifecycle/windows-10-support-has-ended-on-october-14-2025)
Windows 11 업그레이드를 권장합니다. 조건을 충족한 Windows 10 버전 22H2 소비자 기기는
[소비자 ESU](https://www.microsoft.com/en-us/windows/extended-security-updates)에 등록한
경우에만 Microsoft의 현재 안내 기준 2027년 10월 12일까지 긴급·중요 보안 업데이트를 받을 수
있으며 기능 개선과 기술 지원은 포함되지 않습니다. 향후 기간 변경은 링크한 Microsoft 공식
페이지에서 다시 확인해야 합니다. 상용 ESU와 LTSC의 별도 수명주기는 PCssak Biuja 공식 검증
예정 범위에 포함하지 않습니다.

ARM64 네이티브, Windows S 모드, macOS, Linux는 지원하지 않습니다.
[Microsoft Windows 아키텍처 안내](https://support.microsoft.com/en-us/windows/32-bit-and-64-bit-windows-frequently-asked-questions-c6ca9541-8dce-4d48-0415-94a3faa2e13d)도
Windows 11이 64비트 전용임을 확인합니다.

릴리스 노트에 완료된 검증 결과를 기록하기 전에는 “예정”을 “검증 완료”라고 표시하면 안 됩니다.
검증 뒤에도 지원 범위는 시험한 구성을 뜻할 뿐 모든 기기·파일시스템·드라이버·정책·제3자 앱
조합을 뜻하지 않습니다.

## 사용자도 확인해야 하는 부분

- PCSSAK 공식 제품 페이지 또는 `pcssakinc` 릴리스 저장소에서만 내려받으세요.
- 설치 파일 이름과 SHA-256을 `SHA256SUMS.txt`와 비교하세요.
- SmartScreen, Microsoft Defender나 다른 보안 제품을 끄지 마세요.
- 대체할 수 없는 데이터는 별도의 검증된 백업을 유지하세요.
- 중요하지 않은 작은 로컬 폴더에서 시작하고 모든 미리보기를 확인하세요.
- 일부 성공·거부·중단·되돌리기 뒤에는 작업저널을 확인하세요.
- 재현 가능한 오류는 정해진 양식으로, 악용 가능한 보안 문제는 비공개 경로로 보내주세요.

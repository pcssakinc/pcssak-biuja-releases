# 알려진 한계

[English](KNOWN-LIMITATIONS.md)

## 현재 릴리스

- 공개 GitHub 빌드는 1.0 이전의 무료 Early Access입니다.
- 안정 버전 전에는 기능·화면·번역·정리 규칙·작업저널 형식·지원 위치가 바뀔 수 있습니다.
- 발견되지 않은 오류, 분류 실수, 성능 문제와 호환성 문제가 남아 있을 수 있습니다.
- 1인 개발자가 지원하므로 응답·수정·배포 기한을 보장하지 않습니다.
- PCssak Biuja는 정리 도구입니다. 백업, 버전 관리, 보안 삭제, 백신, 저장장치 복구 또는
  데이터 복구를 보장하는 서비스가 아닙니다.

## Windows 호환성

- 현재 지원 중인 Windows 11 Home/Pro x64가 권장 환경이자 공식 검증 예정 대상입니다. 설치
  파일을 만들거나 실행할 수 있다는 이유만으로 공식 검증이 끝난 것은 아닙니다.
- [Microsoft Windows 아키텍처 안내](https://support.microsoft.com/en-us/windows/32-bit-and-64-bit-windows-frequently-asked-questions-c6ca9541-8dce-4d48-0415-94a3faa2e13d)와
  같이 Windows 11은 64비트 전용이며 32비트 에디션이 없습니다.
- x86 설치기는 32비트 Windows 10 실행과 Windows x64의 32비트 앱 호환 시험이라는 두 가지
  베타 용도로만 제공합니다. Windows 11 32비트 운영체제용 빌드가 아니며 x64보다 시험 범위가
  좁습니다.
- Windows 10 22H2 x64·x86은 PCSSAK의 제한적·최선 노력 베타 호환성만 제공하며 공식 검증
  대상이 아닙니다. 기기가 지원하면 Windows 11로 업그레이드하세요.
- ARM64 네이티브, Windows S 모드, macOS, Linux는 지원하지 않습니다.
- Windows ARM 기기에서 x64 앱을 에뮬레이션으로 실행하는 것은 ARM64 네이티브 지원이 아니며
  현재 공식 검증 대상도 아닙니다.
- 필수 WebView2 Runtime, Windows 업데이트, 보안 정책, 파일시스템, 저장장치 드라이버와
  보안 제품이 시작과 파일 작업에 영향을 줄 수 있습니다.

[Microsoft는 Windows 10 일반 지원을 2025년 10월 14일 종료했습니다](https://support.microsoft.com/en-us/windows/deployment/updates-lifecycle/windows-10-support-has-ended-on-october-14-2025).
조건을 충족한 Windows 10 버전 22H2 소비자 기기는
[소비자 ESU](https://www.microsoft.com/en-us/windows/extended-security-updates)에 등록한
경우에만 Microsoft의 현재 안내 기준 2027년 10월 12일까지 긴급·중요 보안 업데이트를 받을 수
있습니다. 기능 개선과 기술 지원은 포함되지 않으며 향후 기간 변경은 링크한 Microsoft 공식
페이지에서 다시 확인해야 합니다. 상용 ESU 연차와 LTSC 에디션은 Microsoft의 별도 수명주기를
따르며 PCssak Biuja 공식 검증 예정 범위를 넓히지 않습니다.

릴리스 노트에는 해당 버전에서 실제 시험한 조합을 기록해야 합니다. 어떤 릴리스 노트도 모든
Windows 업데이트·기기·드라이브·파일시스템·클라우드 공급자·보안 제품 조합을 대표할 수는
없습니다. PCSSAK의 앱 지원이 Windows 10에서 종료된 운영체제 보안 업데이트나 Microsoft 기술
지원을 대신할 수 없습니다.

## 설치 파일과 업데이트

- 최초 설치 파일에는 Windows Authenticode 서명이 없어 알 수 없는 게시자 또는 Microsoft
  Defender SmartScreen이 나타날 수 있습니다. PCSSAK 공식 경로만 사용하고
  `SHA256SUMS.txt`를 확인하세요.
- 설치를 위해 SmartScreen, Microsoft Defender 또는 다른 보안 제품을 끄지 마세요.
- 앱 내부 업데이트는 PCSSAK Tauri 서명을 검증합니다. 이 서명은 Authenticode와 별개이며 최초
  설치의 평판 경고를 없애지 않습니다.
- 자동 업데이트 확인과 다운로드에는 공식 GitHub 릴리스 접속이 필요합니다. 오프라인 파일
  정리는 이 연결이 없어도 사용할 수 있습니다.
- WebView2 Runtime이 없거나 복구가 필요하면 Windows 또는 설치 프로그램이 Microsoft의
  WebView2 배포 서비스에 연결해야 할 수 있습니다.

## 정리 판단과 미리보기

- PCssak Biuja는 앱에서 확인할 수 있는 지원 규칙과 메타데이터만 사용할 수 있습니다. 사용자가
  파일에 부여한 개인적 의미, 계약상 보관 기준, 기록 보존 정책이나 선호 프로젝트 구조까지 알 수
  없습니다.
- 미리보기는 특정 시점의 계획입니다. 미리보기 뒤 실행하기 전에 다른 프로그램, 동기화 도구,
  사용자 작업이나 파일시스템 이벤트가 파일을 바꿀 수 있습니다.
- 모든 미리보기를 직접 확인하세요. 중요하지 않은 작은 폴더에서 시작하고 의도와 맞지 않는 규칙을
  바로잡으세요.
- 미리보기가 성공해도 모든 작업의 실행 성공을 보장하지 않습니다. 권한, 잠금, 여유 공간,
  연결이 끊긴 저장소, 경로 제한, 새로 만들어진 대상 때문에 안전하게 거부되거나 일부만 완료될
  수 있습니다.

## 덮어쓰기 금지와 영구 삭제 금지의 범위

- PCssak Biuja는 기존 대상 파일을 의도적으로 덮어쓰지 않고 사용자 파일을 영구 삭제하지 않도록
  설계됐습니다.
- 이 장치는 PCssak Biuja가 수행한 작업에 적용됩니다. 다른 앱, 클라우드 동기화 공급자, 사용자의
  직접 작업, 하드웨어 장애, 악성코드나 운영체제 문제가 파일을 변경·삭제하는 것까지 막을 수는
  없습니다.
- 충돌·권한 문제·변경된 대상으로 인해 작업을 거부하는 것은 안전을 위한 결과이며 수동 확인이
  필요할 수 있습니다.
- Windows 접근 제어를 우회하거나 운영체제·애플리케이션 보호 폴더를 정리하는 용도가 아닙니다.

## 작업저널과 되돌리기

- 작업저널은 지원되는 완료 작업을 로컬에 기록합니다. 파일 사본이 아니며 백업도 아닙니다.
- 되돌리기는 다른 파일을 덮어쓰지 않고 여전히 안전하게 역작업할 수 있는 저널 기록으로
  제한됩니다.
- 현재 파일이 기록 상태와 다르거나, 원본·대상이 바뀌었거나, 경로가 이미 사용 중이거나,
  드라이브가 없거나, 파일이 잠겼거나, 권한이 바뀌었거나, 외부 프로그램이 파일을 이동·삭제하면
  되돌리기를 사용할 수 없거나 거부될 수 있습니다.
- 지원되는 휴지통 작업을 되돌리려면 해당 항목이 Windows 휴지통에 남아 있어야 합니다. 휴지통,
  Windows, 정리 도구나 사용자가 항목을 제거한 뒤에는 PCssak Biuja가 복원할 수 없습니다.
- 여러 파일 계획은 일부 작업을 완료한 뒤 중단될 수 있습니다. 모든 작업이 한꺼번에 성공하거나
  실패한다고 생각하지 말고 작업저널과 결과 보고를 확인하세요.
- 되돌리기가 있어도 대체할 수 없는 데이터는 별도의 검증된 백업을 유지하세요.

## 위치와 파일시스템 동작

- 시스템 보호 폴더, 프로그램 설치 폴더, 다른 사용자 프로필, 네트워크 공유, 이동식 저장장치,
  클라우드 동기화 폴더, 암호화 위치, 링크·정션·재분석 지점, 특수한 긴 경로, 다른 앱에서 열린
  파일은 특별한 주의가 필요하며 지원되지 않거나 거부될 수 있습니다.
- 클라우드 동기화 폴더 안에서 파일을 이동하면 공급자가 다른 기기에도 변경을 전파할 수 있습니다.
  PCssak Biuja는 공급자의 별도 기록을 통제하거나 되돌리지 않습니다.
- 네트워크와 이동식 저장장치는 미리보기·실행·되돌리기 사이에 연결이 끊길 수 있습니다.
- 파일시스템 시간, 권한, 대체 데이터나 앱별 참조는 눈에 보이는 파일 이름 밖의 영향을 가질 수
  있습니다. 실제 업무 자료에 사용하기 전에 관련 흐름을 시험하세요.

- 파일 정리 자체는 100% 로컬 처리합니다. 파일 이름·경로·미리보기·정리 판단·작업저널·
  되돌리기는 기기에 머물며 정리를 위해 사용자 파일이나 내용을 PCSSAK으로 업로드하지 않습니다.
- 공개 Early Access에는 PCSSAK 계정, 광고, 사용 분석, 추적, 클라우드 파일 색인, 자동 오류
  업로드가 없습니다.
- 네트워크 예외는 공식 GitHub의 서명된 업데이트 확인·다운로드와 필요한 경우 Microsoft
  WebView2 설치·복구입니다.
- 공개 버그 제보는 로컬 처리가 아닙니다. 실제 전체 경로, 폴더 목록, 작업저널, 파일 내용,
  고객 자료나 비밀정보를 GitHub Issue에 붙여 넣지 마세요.

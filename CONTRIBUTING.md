# Issue and Contribution Guidelines / 이슈·기여 원칙

This repository distributes proprietary PCssak Biuja binaries and public documentation and
tracks user reports. It is not the application source repository. The product source is private
and proprietary, and source-code pull requests for PCssak Biuja are not accepted here.

Documentation corrections, reproducible bug reports, and focused feature requests are welcome.
For a documentation change, open an issue first when the intended wording or product behavior
is uncertain. A documentation pull request does not grant rights to or access to the private
application source.

## Responsible reports

- Search existing issues before opening a new one.
- Reproduce the behavior on the latest release when practical.
- Describe the user problem, expected result, actual result, and minimal steps.
- Use a synthetic folder and filenames instead of real personal or customer data.
- Remove full paths, account names, secrets, file contents, directory listings, and confidential
  screenshots.
- State whether the behavior happened in preview, plan execution, work journaling, or rollback.
- Use the private route in [SECURITY.md](SECURITY.md) for exploitable details.
- Remain available for reasonable clarification and retesting questions.

AI may help prepare a report or documentation correction, but the submitter must personally
verify every factual claim, reproduction step, and proposed wording. Do not submit speculative,
duplicate, bulk-generated, or untested issues.

Do not upload third-party files, code, text, screenshots, or other material unless you have the
right to share it under terms appropriate for a public GitHub repository. By contributing
documentation, you confirm that the contribution is your own or properly licensed; it does not
change the proprietary license of PCssak Biuja.

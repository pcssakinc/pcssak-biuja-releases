# Support / 고객지원

PCssak Biuja is maintained by a solo developer. Reports are prioritized by security impact,
file-loss risk, number of affected users, and reproducibility rather than arrival order. Free
Early Access support has no guaranteed response or fix deadline.

## Before reporting

1. Confirm the PCssak Biuja version, installer source, Windows edition and build, and x64 or x86
   architecture.
2. Review the
   [latest release notes](https://github.com/pcssakinc/pcssak-biuja-releases/releases/latest)
   and [known limitations](docs/KNOWN-LIMITATIONS.md).
3. Reproduce the problem with a small synthetic folder that contains no personal, customer,
   licensed, or confidential material.
4. Note whether the problem occurred during scanning, preview, applying a plan, journaling,
   rollback, update checking, installation, or startup.
5. Remove personal folder names, account names, full paths, real filenames, document contents,
   tokens, and unrelated directory listings from screenshots, journal excerpts, and logs.

Use the GitHub [bug report form](../../issues/new?template=bug-report.yml) for a reproducible
application defect. Use the [feature request form](../../issues/new?template=feature-request.yml)
for a recurring user problem that PCssak Biuja could solve. General inquiries can be sent to
`pcssakinc@gmail.com`.

For a useful bug report, include:

- expected and actual behavior;
- minimal numbered steps;
- whether the preview and applied result differed;
- whether an existing destination, locked file, cloud-synchronized folder, network location,
  removable drive, or insufficient permission was involved;
- whether rollback was offered, attempted, completed, or refused;
- sanitized error text and the smallest safe evidence.

Do not attach the original affected folder. A synthetic reproduction is safer and usually more
useful. Exploitable security issues must follow [SECURITY.md](SECURITY.md) and must not be posted
publicly.

## Support boundary

- Currently serviced Windows 11 Home/Pro x64 is the recommended and planned official-validation
  target after the documented validation is completed.
- Windows 10 22H2 x64/x86 receives limited, best-effort beta assistance only. Microsoft ended
  general Windows 10 support on October 14, 2025, so PCSSAK cannot provide missing operating-
  system security updates or Microsoft technical support. Upgrade to Windows 11 is recommended.
- The x86 installer is for beta testing on 32-bit Windows 10 and for testing 32-bit-application
  compatibility on x64 Windows. Windows 11 has no 32-bit edition.
- Native ARM64, Windows S mode, macOS, and Linux are unsupported.
- Preview, journal, and rollback are safety controls, not backup or guaranteed data recovery.
- PCSSAK cannot recover a file that was changed or removed outside PCssak Biuja and is not stored
  in another backup.

See Microsoft's
[Windows 10 end-of-support notice](https://support.microsoft.com/en-us/windows/deployment/updates-lifecycle/windows-10-support-has-ended-on-october-14-2025).
Eligible Windows 10 version 22H2 consumer devices receive critical and important security
updates through October 12, 2027 under Microsoft's current guidance only when enrolled in
[Consumer ESU](https://www.microsoft.com/en-us/windows/extended-security-updates); that program
does not add feature improvements or technical support. Check the linked Microsoft page for
future lifecycle changes. Commercial ESU and LTSC lifecycles are separate and do not change the
PCssak Biuja support boundary.

## 한국어

PCssak Biuja는 1인 개발자가 운영합니다. 문의는 접수 순서만이 아니라 보안 영향, 파일 손실
위험, 영향받는 사용자 수와 재현 가능성을 기준으로 우선순위를 정합니다. 무료 Early Access는
응답이나 수정 기한을 보장하지 않습니다.

제보 전에 다음을 확인하세요.

1. PCssak Biuja 버전, 설치 출처, Windows 에디션·빌드, x64 또는 x86 아키텍처를 확인합니다.
2. [최신 릴리스](https://github.com/pcssakinc/pcssak-biuja-releases/releases/latest)와
   [알려진 한계](docs/KNOWN-LIMITATIONS.ko.md)를 확인합니다.
3. 개인정보·고객 자료·라이선스 자료·회사 비밀이 없는 작은 합성 폴더로 재현합니다.
4. 검사·미리보기·계획 실행·작업저널·되돌리기·업데이트 확인·설치·시작 중 어느 단계에서
   생겼는지 기록합니다.
5. 화면·작업저널 일부·로그에서 개인 폴더 이름, 계정 이름, 전체 경로, 실제 파일 이름, 문서
   내용, 토큰과 관계없는 폴더 목록을 제거합니다.

재현 가능한 오류는 GitHub [버그 제보 양식](../../issues/new?template=bug-report.yml), 반복되는
사용자 문제는 [기능 제안 양식](../../issues/new?template=feature-request.yml)을 사용하세요. 일반
문의는 `pcssakinc@gmail.com`으로 보낼 수 있습니다.

오류 제보에는 기대 결과와 실제 결과, 번호를 붙인 최소 재현 절차, 미리보기와 실제 결과가
달랐는지 여부, 기존 대상·잠긴 파일·클라우드 동기화 폴더·네트워크 위치·이동식 드라이브·권한
부족이 관련됐는지 여부, 되돌리기가 표시·시도·완료·거부됐는지 여부와 안전하게 정리한 오류
문구를 포함하면 좋습니다.

실제 문제 폴더 전체를 첨부하지 마세요. 합성 재현 자료가 더 안전하고 대체로 더 유용합니다.
악용 가능한 보안 문제는 공개하지 말고 [보안 정책](SECURITY.md)을 따라주세요.

현재 지원 중인 Windows 11 Home/Pro x64가 권장 환경이자 검증 절차 완료 후 공식 검증 대상입니다.
Windows 10 22H2 x64·x86은 제한적·최선 노력 베타 지원만 제공합니다. x86 설치기는 32비트
Windows 10 베타 시험과 Windows x64의 32비트 앱 호환 시험용이며, Windows 11에는 32비트
에디션이 없습니다. ARM64 네이티브, Windows S 모드, macOS, Linux는 지원하지 않습니다.

[Microsoft는 Windows 10 일반 지원을 2025년 10월 14일 종료했습니다](https://support.microsoft.com/en-us/windows/deployment/updates-lifecycle/windows-10-support-has-ended-on-october-14-2025).
조건을 충족한 Windows 10 버전 22H2 소비자 기기는
[소비자 ESU](https://www.microsoft.com/en-us/windows/extended-security-updates)에 등록한
경우에만 Microsoft의 현재 안내 기준 2027년 10월 12일까지 긴급·중요 보안 업데이트를 받을 수
있고 기능 개선·기술 지원은 포함되지 않습니다. 향후 기간 변경은 링크한 Microsoft 공식
페이지에서 다시 확인해야 합니다. 상용 ESU와 LTSC의 별도 수명주기는 PCssak Biuja 지원 범위를
넓히지 않습니다. Windows 11 업그레이드를 권장합니다.

미리보기·작업저널·되돌리기는 안전 장치이지 백업이나 데이터 복구 보장이 아닙니다. PCssak
Biuja 밖에서 변경·삭제됐고 별도 백업에도 없는 파일은 PCSSAK이 복구할 수 없습니다.
